Chapter 2 Data sources

For the purposes of this project, we worked with files generated during real matches by the game itself. They are called ‘demo’ files and were created by developers to replay matches from any point of view.

We chose to download all demo files from pro matches that occurred in ‘Majors’ (Majors are the most prestigious events of CSGO). Valve – the company that created CSGO – are a sponsor of Majors, hence making it possible to download the data from the game itself.

However, using the game client makes it harder to implement large-scale scraping as we would need for our analysis purposes. Therefore, we used ‘hltv.org’ – a website compiling all pro-level matches – to download the demo files that we needed; ‘hltv.org’ collaborates with event organizers to display real-time statistics and scores, and shares recorded demo files.

All Major matches can be accessed here: https://www.hltv.org/stats/matches?matchType=Majors
From those matches, demos can be downloaded. For example: https://www.hltv.org/matches/2335630/natus-vincere-vs-mousesports-starladder-majorberlin-2019 (under ‘Rewatch’: ‘GOTV Demo’ button).

We downloaded more than 600 demo files, which adds up to more than 90GB of data.
The Demo files (.dem) are raw files, we used a python library (csgo.parser, by Peter Xenopoulos) to parse them in JSON files, resulting in 625 successful conversions and 20GB of pre-processed data. The generated JSON files can be accessed using a Lion Mail account: https://drive.google.com/drive/folders/1e4d7hVlSO20M1fpgbGD6TZd69vGqg2p_?usp=sharing

Nonetheless, the pre-processed data is still far from being clean; some files are corrupted, and in-game technical pauses, warmup rounds, game restarts, etc. can mess up the format and add inexistant rounds. The following Python notebook explains the data structure of a generated JSON file: https://github.com/anhvung/EDAV_Project_Report/blob/main/code/Understanding%20the%20data.ipynb

Now, how can this data bring answers to our questions?
Game analysts and players traditionally watch the demo replays to study teams andplayer behaviors. Data can help automate this process by revealing some players’ habits. For instance, we could look at the buying habits of a team, their preferential positions on maps, the patterns of their grenade throwing, etc.

Data can help understand the performance of players and teams by calculating somemetrics such as accuracy, teamplay flashes, trades, assists, etc. Those metrics can be combined to create a global rating.
